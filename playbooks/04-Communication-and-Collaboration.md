# Communication & Collaboration

Last updated: 2026-03-25

## Purpose
Move things forward with **clarity, fairness, and minimal friction**.

## When to use
- Any collaboration with deliverables, timelines, or dependency risk
- When follow-ups feel awkward
- When you need to set a boundary or reset expectations

## Core philosophy
"I don't force outcomes. I force clarity."

## Operating model
**Clarity → Observation → Adjustment**

### 1) Create clarity
- Define expectations
- Explain why it matters
- Make the path forward obvious

### 2) Allow decision + observe behavior
Watch for:
- Follow-through
- Communication (especially early)
- Ownership and accountability

### 3) Interpret behavior (operationally)
Don't overreact to the event; interpret the decision behind it.

### 4) Adjust accordingly
Adjust: tone, structure, expectations, level of control.

## Decision-based communication (signature move)
Design messages that force a decision point.

Examples:
- "If you foresee any challenges hitting this, I need to know early. Otherwise, I'll assume we're on track."
- "If timing isn't right, no worries—just tell me so we can close the loop."

## Feedback style
Direct + open:
- Anchor reality, invite explanation
- "This isn't working the way we need it to. Walk me through what happened."
- If it repeats, force a capability decision:
- "We've slipped on this a few times now. Can you reliably own this, or do we need to approach it differently?"

## Saying no
Clear, respectful, grounded. No over-explaining.
- "We're not going to pursue this. It doesn't align with what we're focusing on."

## Boundary setting
Calm, direct, forward-looking.
- "I want to reset expectations here so we're aligned going forward."
